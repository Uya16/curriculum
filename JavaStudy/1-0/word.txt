■ SQL
SQL → シークェルまたはシーケルと呼ばれ、データベースを管理するソフトウェアを操作・制御する言語 プログラミング言語とは違いシステム開発はできないが、データベース言語はデータを管理するプログラムを整理・操作できる様にするもの

DB2 → IBM社が提供しているリレーショナルデータベース管理システムの製品群の総称 幅広いプラットフォームに対応しておりSQLを言語として採用している

MySQL → データベース管理システムであり、高い拡張性や強力なセキュリティや様々な機能からの高い信頼性を持っており、人気のシステムである

Maria DB → MySQLから派生したリレーショナルデータベース管理システム 一般ユーザーから見るとMariaDBの方が使いやすいと考えられ、MySQLと同様に拡張性や信頼性が高いと評価されている

Oracle DB → 堅牢性や読み取りの一貫性に優れたデータベース管理システム

PostgreSQL → リレーショナルデータベース管理システムのひとつで、オープンソースのデータベースの為、全て無料で公開されており自由に利用ができる 条件付きで自由に機能を追加したり改修する事もできる

SQLite → ファイルベースのデータベースエンジンであり、ほとんどのSQL構文の実行ができる ロックする範囲をデータベース全体でしか制御できないものでもある

SQL Server → マイクロソフトが開発したデータ管理システムであり、初心者でも扱い易い操作性とWindows OSと相性が良いのが特徴であり、セキュリティの信頼性が高く、機能を拡張したSQLの使用もできる

CRUD → Create(登録)、Read(参照)、Update(更新)、Delete(削除)の頭文字を並べた用語

トランザクション → ユーザーから見た一連の処理のまとまりの事 複数の処理を一つにまとめたもの

コミット → トランザクションの終了時に更新を確定するという意味で使われる 確定や反映という意味

ロールバック → 変更を取り消す（元の状態に戻す）という意味

排他制御（排他ロック） → トランザクションの実行時、そのトランザクションが対象としているデータをロックし、他のトランザクションのアクセスを禁止することによってデータの整合性を確保する ダブルブッキングを防ぐと言ったイメージ

楽観ロック → 滅多な事では同時更新は起きないであろうという楽観的前提をした上での排他制御 更新対象のデータがデータ取得時と同じ状態である事を確認して行う

悲観ロック → 他者が同じデータに頻繁に変更を加えるのではないかという悲観的前提をした上での排他制御 更新対象となるレコードのロックを取得する事で、他のトランザクションから更新されなくなる

実行計画 → SQLを実行する為の計画であり、SQLを実行する為の具体的な方法を計画としてまとめたもの

チューニング → SQL文のパフォーマンスを特定の測定可能かつ達成可能な目標に合う様向上させる為の反復プロセスであり、レスポンス時間の短縮やリソースの使用量を最小限に抑える事を目的とする

パフォーマンス → SQLの処理性能や実行速度

スロークエリ → 時間のかかるSQLの事

クエリログ → 実行されたSQLのクエリのログ
